# Detailed Setup Guide

This guide covers everything you need to know about setting up and configuring Herd MCP with Claude Code.

## Prerequisites

### 1. Install Claude Code

If you haven't already, install Claude Code:

<Tabs>
  <Tab title="macOS">
    ```bash
    brew install claude-code
    ```

    Or download directly from [claude.ai/claude-code](https://claude.ai/claude-code)
  </Tab>
  <Tab title="Linux">
    ```bash
    curl -fsSL https://claude.ai/install.sh | sh
    ```
  </Tab>
  <Tab title="Windows">
    Download the installer from [claude.ai/claude-code](https://claude.ai/claude-code)
  </Tab>
</Tabs>

Verify installation:

```bash
claude --version
```

### 2. Get Your Herd API Key

1. Go to [herd.eco](https://herd.eco)
2. Create an account or sign in
3. Navigate to your dashboard
4. Copy your API key

<Warning>
Keep your API key secure. Never commit it to version control or share it publicly.
</Warning>

## Adding Herd MCP

### Basic Setup

Run this command to add Herd MCP to Claude Code:

```bash
claude mcp add --transport http herd https://api.herd.eco/v1/mcp --header "Authorization: Bearer YOUR_API_KEY"
```

### Understanding the Command

Let's break down what each part means:

| Part | Description |
|------|-------------|
| `claude mcp add` | Add a new MCP server to Claude Code |
| `--transport http` | Use HTTP transport (required for Herd) |
| `herd` | The name you'll use to reference this MCP |
| `https://api.herd.eco/v1/mcp` | The Herd MCP endpoint URL |
| `--header "Authorization: Bearer ..."` | Your authentication header |

### Verify Setup

Check that Herd MCP is configured:

```bash
claude mcp list
```

You should see `herd` in the list of configured MCP servers.

## Configuration Options

### Environment Variables

You can store your API key as an environment variable for security:

```bash
# Add to your ~/.bashrc, ~/.zshrc, or equivalent
export HERD_API_KEY="your_api_key_here"
```

Then use it in the command:

```bash
claude mcp add --transport http herd https://api.herd.eco/v1/mcp --header "Authorization: Bearer $HERD_API_KEY"
```

### Multiple Configurations

If you have different API keys for different projects:

```bash
# Personal use
claude mcp add --transport http herd-personal https://api.herd.eco/v1/mcp --header "Authorization: Bearer $HERD_PERSONAL_KEY"

# Work projects
claude mcp add --transport http herd-work https://api.herd.eco/v1/mcp --header "Authorization: Bearer $HERD_WORK_KEY"
```

## Removing Herd MCP

If you need to remove or reconfigure:

```bash
claude mcp remove herd
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="'MCP not found' error">
    Make sure you spelled the MCP name correctly. Run `claude mcp list` to see configured MCPs.
  </Accordion>

  <Accordion title="Authentication errors">
    Double-check your API key. Make sure there are no extra spaces or quotes. Try regenerating your key in the Herd dashboard.
  </Accordion>

  <Accordion title="Connection timeout">
    Check your internet connection. The Herd API requires an active internet connection.
  </Accordion>

  <Accordion title="Tool not responding">
    Try removing and re-adding the MCP:
    ```bash
    claude mcp remove herd
    claude mcp add --transport http herd https://api.herd.eco/v1/mcp --header "Authorization: Bearer YOUR_API_KEY"
    ```
  </Accordion>
</AccordionGroup>
